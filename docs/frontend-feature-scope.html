Frontend Feature Scope

This document describes the application features page by page. User authentication and user management are excluded because those already exist in the target application.

Dashboard

The dashboard is the main overview page. It should quickly show what needs attention today.

Functions

  • Show active tasks.
  • Show overdue tasks.
  • Show tasks waiting for review.
  • Show open issues.
  • Show at-risk projects.
  • Show website health problems.
  • Show recent activity.
  • Show recent notes.
  • Show quick links to important areas.

The goal is to help the team immediately see what is urgent, what is blocked, and what should be handled next.

Projects

The Projects page manages clients, projects, and their related websites.

Functions

  • View all projects in a table.
  • Search and filter projects.
  • Add a new project.
  • Edit and view project details.
  • Delete a project.
  • Group projects by priority: High, Medium, and Low.
  • Change priority by dragging a project into another priority group.
  • Assign a project to a team member.
  • Store multiple website or domain links per project.
  • Add, edit, and remove website or domain records.
  • Save project changes without closing the edit panel.
  • Import projects in bulk from Google Sheets, CSV, or Excel.
  • Preview imported data before saving.
  • Map imported columns to project fields.
  • Skip columns that should not be imported.
  • Show import success and row-level errors.

Tasks

The Tasks page manages all team tasks.

Functions

  • View tasks in a Kanban board.
  • Use columns: Backlog, In Progress, In Review, and Done.
  • Drag tasks between columns.
  • Filter or switch by project.
  • Add a new task.
  • View and edit task details.
  • Update task status.
  • Update task priority.
  • Update checklist progress.
  • Set start date and due date.
  • Attach links or files.
  • Mark checklist items as done.

Task Card

  • Show task title.
  • Show checklist progress.
  • Show due date.
  • Show status.
  • Show priority using a small color-coded icon.
  • Show view action.
  • Show done action.

Add Task Flow

  • First screen: select client or project.
  • First screen: paste raw task details into a large text box.
  • The pasted details can come from chat messages, notes, documents, emails, or links.
  • Links inside the text should be clickable.
  • Click Organize with AI to structure the task.
  • Second screen: show AI-generated title.
  • Second screen: show AI-generated description.
  • Second screen: show AI-generated checklist.
  • Second screen: show priority, status, date range, and attachments or links.

The AI should organize messy pasted instructions into a clean task that is editable before saving.

Task Detail

The task detail screen should work as both view and edit.

Fields

  • Client or project.
  • Title.
  • Description.
  • Checklist.
  • Progress.
  • Priority.
  • Status.
  • Start date.
  • Due date.
  • Attachments or links.

Functions

  • Edit the title.
  • Edit the description.
  • Check and uncheck checklist items.
  • Add and remove checklist items.
  • Update priority.
  • Update status.
  • Update date range.
  • Remove attachments or links.
  • Save changes.

Task Summary

The Task Summary page gives a structured overview of work by assignee and project.

Functions

  • Filter tasks by status.
  • Group tasks by assignee.
  • Show each assignee as a simple heading.
  • Under each assignee, show a table of projects.
  • Clicking a project expands it.
  • Expanded project shows tasks under that project matching the selected status.

Task Item

  • Show title.
  • Show status.
  • Show priority.
  • Show checklist progress.
  • Show view and edit action.

My Tasks

The My Tasks page shows only the current user's assigned tasks.

Functions

  • View personal tasks in a Kanban board.
  • Use columns: Backlog, In Progress, In Review, and Done.
  • Filter by project.
  • Add a new task.
  • Drag tasks between columns.
  • View and edit task details.
  • Update task progress.
  • Use the same task card design as the main Tasks page.

Website Health

The Website Health page monitors all websites. Each row represents one website or domain.

Functions

  • Show all websites in a quick overview table.
  • Keep websites from the same project or client grouped together.
  • Scan a website.
  • View detailed scan results.
  • Export health reports.
  • Show last scan or update date.
  • Show connector status.

Main Table Columns

  • Website and project.
  • Overall score.
  • Number of pages.
  • Lighthouse average.
  • Technical SEO issue count.
  • Design QA issue count.
  • Website checklist unresolved count.
  • Last scan or update.
  • Connector status.
  • Actions.

Actions

  • View details.
  • Start scan.
  • Export report.

Details Drawer

  • Show overview cards for Pages, Lighthouse, Technical SEO, Design QA, Forms, and Website Checklists.
  • Show priority issues clearly.
  • Show per-page details for Lighthouse, Technical SEO, Design QA, and Forms.

Website Health Checks

  • Page speed and performance.
  • Accessibility.
  • Best practices.
  • SEO score.
  • Technical SEO issues.
  • Layout and design issues.
  • Mobile, tablet, and desktop responsiveness.
  • Broken layouts.
  • Font inconsistencies.
  • Dummy or incorrect content.
  • Wrong names or unrelated content.
  • Forms found on each page.
  • Whether forms appear functional.
  • Whether forms have reCAPTCHA.
  • WordPress version.
  • Plugin list.
  • Plugin update status.
  • Required plugins present or missing.
  • Website users.
  • Password age.
  • Security checklist.

Technical SEO

Technical SEO should work like a checklist. The table should show total SEO issues across all pages.

Checks

  • Missing page title.
  • Duplicate page title.
  • Missing meta description.
  • Duplicate meta description.
  • Missing H1.
  • Multiple H1 tags.
  • Bad heading structure.
  • Missing image alt text.
  • Broken links.
  • Missing canonical URL.
  • Bad indexability settings.
  • Missing sitemap.
  • Missing robots file.
  • Missing schema markup.
  • Poor URL structure.
  • Thin content.
  • Missing social sharing metadata.

Design QA

Design QA checks the visual quality of each page.

Checks

  • Broken layout.
  • Overlapping elements.
  • Bad spacing.
  • Incorrect font size.
  • Inconsistent typography.
  • Elements not aligned.
  • Broken responsive layout.
  • Content cut off.
  • Buttons or sections displaying incorrectly.
  • Dummy content.
  • Incorrect names.
  • Unrelated content.
  • Wrong doctor, team, or business names.
  • Missing sections.
  • Page does not match expected design quality.

A future feature can compare a webpage against a design file.

Forms Check

The forms check should detect forms on the website.

For each form, show

  • Page where the form appears.
  • Form name or detected purpose.
  • Number of fields.
  • Required fields.
  • Whether it appears functional.
  • Whether reCAPTCHA is present.
  • Any visible issues.

Website Checklist

The website checklist should cover WordPress and operational health.

Checks

  • WordPress version.
  • Whether WordPress is updated.
  • Plugin list.
  • Plugin update status.
  • Required plugins present.
  • Required plugins missing.
  • Website users.
  • Password last updated date.
  • Old or stale passwords.
  • Security headers.
  • HTTPS.
  • Mixed content.
  • Exposed sensitive files.
  • Basic hardening checks.

Functions

  • Edit issue.
  • Delete issue.
  • Change issue status.
  • Assign issue to project or website.

Issue Fields

  • Title.
  • Description.

Activity Logs

Activity logs help debug and audit what happened.

Groups

  • Website activity.
  • System or team activity.

Columns

  • Name.
  • IP address.
  • Action.
  • Date and time.

Functions

  • View activity logs.
  • Filter by project.
  • Filter by website.
  • Filter by action.
  • Filter by date range.
  • Paginate results.
  • Show severity or status color.

Logged Actions

  • Project created, updated, or deleted.
  • Website or domain added, updated, or deleted.
  • Task created, updated, moved, or deleted.
  • Credential created, updated, copied, or deleted.
  • Website scan started, completed, or failed.
  • Connector paired or disconnected.
  • Settings changed.
  • Notification settings changed.
  • AI prompt changed.

My Notes

My Notes is a personal notes page similar to Google Keep.

Functions

  • Create note.
  • Edit note.
  • Delete note.
  • Search notes.
  • Show notes as cards.
  • Show created or updated date under each card.
  • Show empty state when there are no notes.

Future feature: add reminder date and time, then send a reminder notification.

Settings

The Settings page controls application behavior.

Sections

  • General settings.
  • Notification settings.
  • Email connector.
  • AI settings.

General Settings

  • Workspace or app name.
  • Time zone.
  • Time zone options: Dubai, London, and Manila.

Notification Settings

  • Turn notifications off.
  • Send by email.
  • Send by Discord.
  • Send by both email and Discord.

Daily User Summary

  • Sends summary of what was completed.
  • Has a time field.

Pre-Shift Briefing

  • Sends todos, priorities, and manager notes before shift starts.
  • Has a time field.

Email Connector

  • Connect email provider.
  • Disconnect email provider.
  • Test email sending.
  • Keep setup simple.
  • Do not show advanced settings if not needed.

AI Settings

  • AI model.
  • Task organizer system prompt.
  • Task organizer user prompt template.
  • Prompt must be required.
  • If no prompt exists, task organization should not run.

Functions

  • Validate connection.

  • Send plugin update status.

Reports and Exporting

Reports should not be a standalone main page. Reports belong inside Website Health.

Export Function

  • Let the user choose website.
  • Let the user choose which reports to include.
  • Combine selected reports into one file.
  • Export without showing scan result previews inside the modal.

Report Types

  • Lighthouse.
  • Technical SEO.
  • Design QA.
  • Forms.
  • Website checklist.
  • Security checklist.

Global UI Behavior

  • Use a clean, professional, light-theme interface.
  • Use compact tables.
  • Use white table containers.
  • Keep filters in one row when possible.
  • Filters should be searchable.
  • Search field should focus automatically when filter dropdown opens.
  • Use toast notifications instead of inline alerts.
  • Toasts should appear at the top center.
  • Toasts should be color-coded: success, warning, error, and info.
  • Buttons should have consistent border radius.
  • Avoid overly rounded pill buttons unless needed.
  • Avoid nested cards.
  • Avoid cluttered layouts.
  • Use reusable components for repeated patterns.
  • Keep table actions clean, preferably dropdown actions where there are multiple choices.

Important Product Rules

  • Avoid dummy data once real data exists.
  • Keep credentials hidden unless explicitly copied or viewed.
  • Track sensitive actions in activity logs.
  • Every form should validate inputs.
  • Bulk imports should always preview before saving.
  • AI-generated content should be editable before saving.
  • Website health should show quick summary first, then detailed breakdown.
  • Task details and Add Task should use the same structure.
  • My Tasks and Tasks should use the same task card design.
